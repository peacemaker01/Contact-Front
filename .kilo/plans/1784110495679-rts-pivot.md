# RTS Pivot Plan - Real-Time Strategy Architecture

## Architecture Shift Summary
- **From**: Turn-based (staged actions → commit → resolve)
- **To**: Real-time continuous game loop with live actions/movements
- **Satellite Imagery**: New Game uses randomized real-world satellite terrain
- **Scenario Builder**: Map-based visual editor with unit placement tools

## RTS Milestone Structure (Replaces Turn-Based Plan)

### Milestone 0 — Repo hygiene
1. Rotate the OpenRouter key committed in the old Python repo's `newach/.env`
2. `.gitignore` covering `target/`, `build/`, `.idea/`, `*.class`
3. Add pre-commit secret scanner (`gitleaks`)

### Milestone 1 — Core rule-based engine (RTS-adapted)
- Movement cost, LOS/recon, combat resolution, suppression, stances
- `sealed interface Action` types (MoveAction, AttackAction, etc.)
- **Acceptance**: headless JUnit suite before GUI

### Milestone 2 — Data-driven unit roster
- Units/weapons in JSON config (infantry, armor, recon, artillery, drone, engineer, AA, logistics)
- Combat uses damage_class vs armor_class lookup table
- **Acceptance**: new unit/weapon = only JSON changes

### Milestone 3 — Satellite terrain INTEGRATION (CORE CHANGE)
- **New Game default**: Download satellite imagery for random real-world location
- Fallback to procedural terrain when API key unavailable
- Async loading screen during download/process
- Cache in `cache/maps/` directory
- **Affected**: `SatelliteImageProcessor.java`, `App.java` (`handleLocationSelection`)

### Milestone 4 — Scenario Editor (MAP-BASED, CORE CHANGE)
- Visual unit placement on map with preview ghosts
- Side toolbar with draggable unit templates by faction
- Click-to-place with faction selection
- Player can control placed enemy units (single-player coop)
- Save format: scenario JSON with unit positions + terrain data
- **Affected**: `ScenarioBuilder.java` → rename to `ScenarioEditor.java`, `MapView.java`

### Milestone 5 — REAL-TIME AI LOOP (CORE CHANGE)
- Fixed-timestep (500ms ticks) game loop in `TacticalEngine`
- Continuous `AiTurn.run()` (skip for player-controlled units)
- No "End Turn" button — pause/resume toggle instead
- Game time in seconds, not turns
- **Affected**: `TacticalEngine.java`, `GameState.java` (remove turn counter)

### Milestone 6 — JavaFX tactical HUD (RTS-adapted)
- Top bar: mission name, elapsed time, faction
- Map area: Canvas with terrain + unit symbology
- Right panel: selected unit status, action buttons
- Bottom: event log (combat/intel/orders)
- **Interaction**: Drag-select, right-click context orders, hotkeys 1-9, pan/zoom
- **Removed**: "End Turn" button

### Milestone 7 — Save/load (state-based)
- Save: seed + terrain data + unit positions + casualty state
- Load: reconstruct exact state
- **Acceptance**: saved game reloads identically

### Milestone 8 — Packaging
- `jpackage` bundles JRE
- **Acceptance**: launches on clean machine

## Key Design Decisions

| Decision | Rationale |
|---|---|
| 500ms fixed timestep | Deterministic, testable, real-time feel |
| Satellite for New Game | Real-world terrain, visual plausibility |
| Map-based editor | Visual placement, intuitive unit composition |
| Player-controlled enemies | Single-player cooperative play |
| Doctrine micro-tactics | Distinct faction personalities |
